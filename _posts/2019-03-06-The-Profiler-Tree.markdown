# 序言

性能这个话题，经常令人谈虎色变。因为我们经常会对主业务流程进行性能压测，但是主业务流程的代码里可能会涉及各种数据库增删改查，第三方系统RPC调用，消息发送等操作，当发现性能瓶颈的时候我们无法很快的定位到底是哪个具体的操作耗时高。为了解决这个头疼的问题，**调用树**这个性能问题排查神器就可以排上用场。
